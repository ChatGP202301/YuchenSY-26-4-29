## 🔒 7层防护详解

### 第1层：Google reCAPTCHA v3（隐形验证）

**工作原理**：
- 完全隐形，用户无需操作（无需点击"我不是机器人"）
- Google AI 在后台分析用户行为
- 给每个提交评分（0.0 - 1.0）
  - 0.0 = 100% 机器人
  - 1.0 = 100% 人类
- 分数低于阈值 → 自动拒绝

**如何配置**：
1. 访问：https://www.google.com/recaptcha/admin/create
2. 选择：**reCAPTCHA v3**
3. 添加域名：`expresswater.cn` 或 `yuchensy.com`
4. 获取 **Site Key**（类似：`6LcXXXXXXXXXXXXXXXXXXXXXXXXXXX`）
5. 在所有 contact.html 中查找并替换：
   ```
   YOUR_RECAPTCHA_SITE_KEY
   ```
   改为您的实际 Site Key

**重要**：
- ⚠️ 如果不配置，reCAPTCHA 会被跳过，但其他6层仍然有效！
- ✅ 配置后，垃圾邮件拦截率可达 99.9%

### 第2层：蜜罐字段（Honeypot）

**工作原理**：
- 在表单中插入一个隐藏字段 `name="website"`
- CSS 使其完全不可见（position: absolute; left: -9999px）
- 真人看不到也不会填写
- 机器人会自动检测并填写所有字段
- 如果该字段有值 → 100% 是机器人 → 静默拒绝

**拦截效果**：
```
真人用户：看不到此字段，不填写 ✅ 通过
简单机器人：自动填写所有字段 ❌ 被拦截（静默跳转到 thank-you.html）
```

**优势**：
- 完全不影响用户体验
- 对简单机器人100%有效
- 无需额外配置

---

### 第3层：时间戳验证

**工作原理**：
- 记录表单加载时间：`formLoadTime`
- 计算用户填写耗时：`fillTime = 提交时间 - 加载时间`
- 验证规则：
  - **太快**（< 3秒）：机器人可以瞬间填写，人类需要时间阅读
  - **太慢**（> 10分钟）：可能是机器人暂停脚本或会话超时

**拦截效果**：
```
提交耗时 0.5秒 → ❌ 太快，机器人
提交耗时 5秒 → ✅ 正常
提交耗时 15分钟 → ❌ 超时，会话过期
```

**用户提示**：
- 太快：显示 "Please take your time to fill out the form carefully"
- 太慢：显示 "Form session expired. Please refresh and try again"

---

### 第4层：行为分析（人类行为检测）

**追踪的行为**：
1. **鼠标移动**（mouseMovements）
   - 人类：不规则移动轨迹
   - 机器人：无鼠标移动或直线移动

2. **键盘输入**（keyPresses）
   - 人类：有打字节奏，有删除修改
   - 机器人：瞬间填满所有字段

3. **焦点事件**（focusEvents）
   - 人类：按Tab或点击切换字段
   - 机器人：直接设置value，无焦点切换

4. **滚动事件**（scrollEvents）
   - 人类：阅读页面时会滚动
   - 机器人：直接定位到表单

5. **点击事件**（clickEvents）
   - 人类：点击字段、按钮
   - 机器人：脚本触发，无点击

6. **触摸事件**（touchEvents，移动设备）
   - 人类：手指触摸屏幕
   - 机器人：无触摸事件

**判断逻辑**：
```javascript
最少交互次数 = 5次（桌面）或 1次（移动设备）
总交互 = 鼠标 + 键盘 + 触摸

if (总交互 < 最少交互) {
  → ❌ 机器人（静默拒绝）
} else {
  → ✅ 人类
}
```

**拦截效果**：
```
纯脚本填写（无任何交互）→ ❌ 被拦截
真人操作（鼠标+键盘）→ ✅ 通过
```

### 第5层：提交频率限制

**工作原理**：
- 使用 `localStorage` 记录上次提交时间
- 规则：同一浏览器 60秒内只能提交 1次
- 超过限制 → 显示等待时间

**拦截效果**：
```
第1次提交：✅ 允许，记录时间戳
30秒后第2次提交：❌ "Please wait 30 seconds..."
70秒后第2次提交：✅ 允许
```

**防止**：
- 批量垃圾邮件攻击
- 同一用户重复提交
- 恶意测试表单

### 第6层：客户端指纹识别

**收集的信息**：
```javascript
{
  userAgent: "Mozilla/5.0...",           // 浏览器标识
  language: "en-US",                     // 语言
  platform: "Win32",                     // 操作系统
  screenResolution: "1920x1080x24",      // 屏幕分辨率
  timezone: "America/New_York",          // 时区
  timezoneOffset: -300,                  // 时区偏移
  plugins: 5,                            // 浏览器插件数量
  cookieEnabled: true,                   // Cookie是否启用
  webGL: true,                           // WebGL支持
  canvas: true                           // Canvas支持
}
```

**用途**：
- 生成唯一设备指纹
- 识别可疑特征（如无头浏览器）
- 发送到服务器用于后续分析
- 可以追踪同一设备的重复提交

**异常特征**：
```
正常浏览器：plugins > 0, webGL = true, canvas = true
无头浏览器：plugins = 0, webGL = false, 可疑UA
```

---

### 第7层：自动化工具检测

**检测的工具**：

1. **Selenium / WebDriver**
   ```javascript
   if (navigator.webdriver === true) → 机器人
   ```

2. **PhantomJS**
   ```javascript
   if (window._phantom || window.callPhantom) → 机器人
   ```

3. **Nightmare.js**
   ```javascript
   if (window.__nightmare) → 机器人
   ```

4. **Headless Chrome**
   ```javascript
   if (userAgent.includes('HeadlessChrome')) → 机器人
   ```

5. **无插件异常**
   ```javascript
   if (navigator.plugins.length === 0) → 可疑
   ```

**拦截策略**：
- 检测到自动化标识 → 记录日志
- 可以选择直接拒绝或仅标记
