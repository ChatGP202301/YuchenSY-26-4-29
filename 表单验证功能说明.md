# 📋 表单验证功能说明

## ✅ 已实现的验证功能

### 1. **所有字段必填**（除了 Company Name）

| 字段 | 要求 |
|------|------|
| Your Name | 必填，最少2个字符 |
| Company Name | 可选 |
| Email Address | 必填，必须是有效邮箱格式 |
| Phone / WhatsApp | 必填，必须是国际格式（+国家码） |
| Country | 必填，最少2个字符 |
| Your Inquiry | 必填，最少10个字符 |

---

### 2. **邮箱验证**（严格）

**验证规则**：
- ✅ 符合 RFC 5322 邮箱标准
- ✅ 必须包含 `@` 和有效域名
- ✅ 域名必须有顶级域（如 .com, .net）
- ✅ 拒绝常见假邮箱域名（test.com, example.com, fake.com）

**示例**：
- ✅ 有效: `john@company.com`, `sales@express-water.cn`
- ❌ 无效: `john@test`, `abc@example.com`, `test@fake.com`

**实时反馈**：
- 用户离开邮箱字段时自动验证
- 无效邮箱会显示红色边框和错误提示

---

### 3. **国际电话号码验证**（使用 libphonenumber-js）

**验证规则**：
- ✅ 必须以 `+` 开头（国际格式）
- ✅ 自动识别国家码
- ✅ 验证号码长度和格式是否符合该国标准
- ✅ 检查号码是否为真实有效的号码

**支持的格式**（示例）：

| 国家 | 格式示例 | 国家码 |
|------|---------|--------|
| 美国 | +1 555-1234 | +1 |
| 中国 | +86 138-0000-0000 | +86 |
| 英国 | +44 20 1234 5678 | +44 |
| 德国 | +49 30 12345678 | +49 |
| 法国 | +33 1 23 45 67 89 | +33 |
| 日本 | +81 3-1234-5678 | +81 |
| 韩国 | +82 2-1234-5678 | +82 |
| 印度 | +91 11 1234 5678 | +91 |
| 巴西 | +55 11 91234-5678 | +55 |
| 澳大利亚 | +61 2 1234 5678 | +61 |

**智能功能**：
- 🎯 **自动识别国家**：输入电话号码后，如果国家字段为空，会自动填充国家名称
- ⚡ **实时验证**：边输入边验证，输入完整号码后立即反馈
- 🌐 **全球覆盖**：支持全球200+个国家和地区的电话号码格式

**错误提示**：
- 未加国家码：显示 "Invalid phone format. Include country code (e.g., +1 for US)"
- 号码无效：显示 "Please enter a valid phone number"

---

### 4. **实时错误提示**

**用户体验**：
1. 用户填写字段并离开 → 立即验证并显示错误（如有）
2. 错误字段有红色边框
3. 字段下方显示具体错误消息（红色文字）
4. 用户重新输入正确内容 → 错误消息自动消失，边框变绿
5. 提交前全面验证所有字段

**多语言错误消息**：
- 英语: "This field is required"
- 西班牙语: "Este campo es obligatorio"
- 法语: "Ce champ est obligatoire"
- 德语: "Dieses Feld ist erforderlich"
- （根据页面语言自动切换）

---

### 5. **提交前最终验证**

用户点击 "Send Inquiry" 按钮时：

1. ✅ 检查所有必填字段
2. ✅ 验证邮箱格式
3. ✅ 验证电话号码真实性
4. ✅ 如有错误，显示所有错误并聚焦到第一个错误字段
5. ✅ 所有验证通过后才允许提交

## 🌐 技术实现

  - CDN: `https://cdn.jsdelivr.net/npm/libphonenumber-js@1.10.51/bundle/libphonenumber-min.js`

**HTML5 验证**：

**JavaScript 验证**：

- 正则表达式邮箱验证
- libphonenumber-js 电话验证
- 事件监听（blur, input, submit）

## ⚠️ 重要提示

### 服务器端验证
虽然客户端验证可以防止大部分无效提交，但**仍然需要服务器端验证**作为最后一道防线，因为：
- 用户可以禁用 JavaScript
- 恶意用户可以绕过客户端验证

**建议**：
1. Formsubmit.co 会做基本的邮箱验证
2. 如果您有自己的后端，也应实施相同的验证规则

### 隐私保护
- 电话号码验证在客户端进行，不会发送到第三方服务器
- libphonenumber-js 是纯客户端库，所有验证都在用户浏览器中完成
